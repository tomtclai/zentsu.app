{%- comment -%} head-common.html: shared head contents for every Zentsu marketing page. Each caller owns the opening and closing head tags. This include writes the scaffolding that used to be hand-pasted into each page: charset/viewport, icons, fonts, the shared stylesheet, the , the description meta, canonical, and the full Open Graph / Twitter card block. Per-page values come from front-matter, so each page declares its title and description exactly once instead of repeating them 3 to 4 times. Front-matter each page sets (all optional except title + description): title: <title>, og:title, twitter:title (required) description: meta description, og/twitter description (required) og_description: shorter social blurb, if the meta description is too long for a card (falls back to `description`) twitter_description: distinct Twitter-card blurb (falls back to og_description, then description) canonical: absolute canonical URL. Falls back to the page's own URL. og_site_name: "Zentsu LLC" (default) or "Bench" on Bench-scoped pages og_type: "website" (default) or "article" og_image: absolute OG image URL (default: the studio zentsu-og.png) og_image_alt: alt text for the OG image page_stylesheet: extra per-page stylesheet path (e.g. /bench.css). The shared.css base is always loaded first. Anything a page needs beyond this (JSON-LD blocks, {% feed_meta %}, theme-color) it adds after this include and before its closing head tag. This file only owns the scaffolding that was genuinely duplicated. seo_managed_externally: set true on pages that emit their own title/OG via the jekyll-seo-tag plugin's {% seo %} (i.e. blog posts, via _layouts/post.html).
