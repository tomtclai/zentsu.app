# Product Marketing Context

## Company

Zentsu LLC builds small, native, local-first software for Apple platforms.

## Current Product Focus

Bench is the lead product. It is a native macOS developer toolbox for inspecting whatever the user just copied: JSON, JWTs, regexes, hashes, colors, UUIDs, timestamps, text, images, and video files.

## Positioning

Bench is the offline Mac toolbox that opens the right developer tool from your clipboard.

The strongest wedge is not "many tools in one app." It is "paste once, Bench already knows which tool you need." That makes Bench feel faster and more native than searching for a web tool, opening a tab, pasting data, and wondering where the data went.

## Primary Buyer

Developers, indie hackers, support engineers, QA engineers, and technical founders who repeatedly inspect copied data while debugging APIs, logs, tokens, screenshots, release assets, and configuration.

## Core Differentiators

- Smart Paste: rule-based local detection routes copied content to the right tool.
- Offline by default: no account, no telemetry, no network calls for tool data.
- Native Mac app: Swift / SwiftUI, single-window workflow, fast switching.
- Free download with an optional Bench Pro unlock: no subscription; the downloaded version keeps working.
- Privacy trust: sensitive payloads stay on the user's Mac.

## Messaging Priorities

1. Make the clipboard workflow obvious above the fold.
2. Prove privacy early, before asking for trust.
3. Use concrete examples: JSON, JWT, regex, hex color, hash.
4. Avoid over-selling AI. The "smart" part is deterministic and rule-based.
5. Keep the purchase model simple: free download; optional one-time Bench Pro in-app purchase; no subscription.

## Current Conversion Goal

Bench has been submitted to the Mac App Store. It is a free download with an optional non-consumable Bench Pro in-app purchase. Until the App Store URL is live, the primary website conversion should be email capture for launch/release notes. App Store CTAs should only become primary once the app has a real App Store URL.

## Best Bang-For-Buck Website Change

Replace empty or premature "Download/View on the Mac App Store" CTAs with a launch-notes CTA that scrolls to the email form, framed around "submitted to the Mac App Store." This fixes trust, gives visitors a real action, and builds a launch audience.

## Tool Page Strategy

Tool pages should be long-tail search landing pages. Each page should match high-intent queries like "offline JSON formatter for Mac" or "JWT decoder Mac no upload." The page should quickly show:

- what the tool does;
- why local/offline matters;
- how Smart Paste detects the content;
- related tools in Bench;
- a CTA to get Bench launch notes until the App Store link is live.
